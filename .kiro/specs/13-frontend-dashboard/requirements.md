# Dashboard 与 Trading 页面需求

## 概述
实现核心用户页面：Dashboard 仪表盘和 Trading 交易监控，包括 WebSocket 实时数据推送。

## 功能需求

### FR-1: WebSocket 集成
- FR-1.1: 实现 WebSocket 连接管理器
- FR-1.2: 支持自动重连机制
- FR-1.3: 支持多频道订阅 (trading, risk, state, market)
- FR-1.4: 实现心跳检测

### FR-2: Dashboard 页面
- FR-2.1: 账户余额卡片（实时更新）
- FR-2.2: 今日 PnL 卡片
- FR-2.3: 当前持仓卡片
- FR-2.4: 系统状态卡片
- FR-2.5: 收益曲线图（7天/30天切换）
- FR-2.6: 最近订单列表（5条）
- FR-2.7: 风控指标概览

### FR-3: Trading 页面
- FR-3.1: K线图组件（支持多周期）
- FR-3.2: 当前持仓详情
- FR-3.3: 活跃订单列表
- FR-3.4: 历史订单列表（分页）
- FR-3.5: 订单撤销功能
- FR-3.6: 交易标记在 K线图上显示

### FR-4: 数据展示
- FR-4.1: 数字格式化（金额、百分比）
- FR-4.2: 颜色编码（盈利绿色、亏损红色）
- FR-4.3: 时间格式化（本地时区）
- FR-4.4: 加载状态和错误处理

## 非功能需求

### NFR-1: 实时性
- WebSocket 数据延迟 < 100ms
- 断线重连 < 5s

### NFR-2: 移动端适配
- Dashboard 卡片垂直排列
- K线图支持全屏模式
- 订单列表支持下拉刷新
