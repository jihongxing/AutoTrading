## **策略筛选优先级清单**

> **核心认知**：在高效市场中，不存在"强信号"（p < 0.01, 胜率 > 60%）。系统价值在于组合多个弱信号（p < 0.20, 胜率 52-55%），通过严格风控和大样本累积实现长期稳定盈利。

### 1️⃣ 核心筛选维度（基于弱信号组合哲学）

|序号|维度|说明|权重/优先级|
|---|---|---|---|
|1|**统计显著性**|p < 0.20 即可接受（TIER 1/2），不追求 p < 0.01|高|
|2|**方向性倾向**|胜率 52-55% 即为有效信号，不追求 > 60%|高|
|3|**风险可控**|策略最大回撤、单次下单风险可接受|高|
|4|**数据可获取性**|所需数据公开、延迟低、质量稳定|高|
|5|**组合互补性**|与现有信号低相关，提供独立视角|高|

### 2️⃣ 次级筛选维度（优化性能）

|序号|维度|说明|权重/优先级|
|---|---|---|---|
|5|**信号频率**|高频/中频/低频匹配系统目标交易频率|中|
|6|**执行复杂度**|是否容易在内核执行层实现，是否需要复杂对冲或额外模块|中|
|7|**可扩展性**|未来可接入自学习模块动态优化权重|中|
|8|**市场适用性**|是否适用于不同市场/不同周期|中|

### 3️⃣ 候选策略来源

1. 高频微结构类
    
    - 资金费率套利、三角套利、订单簿失衡、大单冲击
        
2. 中频方向类
    
    - 清算密度、清算热力图、资金费率极端、持仓量异常
        
3. 链上/宏观类
    
    - 巨鲸转账、交易所净流入/流出
        
4. 机器学习增强类（谨慎）
    
    - 特征工程 + LightGBM/XGBoost
        
    - 强化学习（风险高，不作为初期策略）

## **策略验证流程**

```mathematica
策略候选池 (Hypothesis Pool)
          │
          ▼
   数据收集与整理
   - 历史K线、资金费率、成交量、订单簿、链上数据
          │
          ▼
   初步筛选
   - 数据完整性
   - 计算方向性指标 (P(up)/P(down))
   - 检查正期望
          │
          ▼
   统计检验/回测验证
   - 计算回测收益、最大回撤
   - p-value / Cohen's d / KL Divergence
   - 高频策略可加短期窗口验证
          │
          ▼
   策略评分与排序
   - 核心指标：方向性、正期望、风险控制
   - 次级指标：频率、复杂度、扩展性
          │
          ▼
   核心策略候选筛选
   - 选出 1~N 个方向明确、盈利可控策略
          │
          ▼
   封装为策略对象/证人
   - 指定发言权（核心/辅助/风控）
   - 绑定内核执行规则
          │
          ▼
   接入内核执行层
   - 三权分离 + 风控 + 仓位管理
          │
          ▼
   自学习模块持续优化
   - 实盘表现收集
   - 动态调整策略权重或剔除无效策略

```

### 🔑 核心原则（基于弱信号组合哲学）

1. **接受弱信号**
   - ✅ TIER 1: p < 0.05, 胜率 52-55%（核心证人）
   - ✅ TIER 2: p < 0.20, 胜率 51-53%（辅助证人）
   - ❌ 不追求 PASS: p < 0.01, 胜率 > 60%（理论上不存在）

2. **组合优于单一**
   - 单个弱信号：52% 胜率
   - 3 个组合：55-58% 胜率
   - 5 个组合：58-62% 胜率

3. **风控优先于策略**
   - 即使所有策略失效，只要风控有效，系统依然可以生存
   - 目标：年化 20-50%，夏普 > 1.5，生存 > 5 年

4. **大样本累积**
   - 弱信号需要大量交易样本才能显现优势
   - 单笔交易结果不重要，长期统计优势才是核心

5. **持续迭代**
   - 验证完成后策略进入内核，自学习模块持续优化权重
   - 表现不佳的证人降权或剔除，保持系统健康
